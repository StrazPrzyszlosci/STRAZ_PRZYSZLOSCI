## Architektura docelowa (poprawiona)

```
Discord API (Gateway WebSocket)
    ↕ ws (discord.js)
Discord Bot (discord_bot.mjs, Render.com)  ← ~200 linii, cienki gateway
    ↕ HTTP POST /api/discord/webhook
Cloudflare Worker (worker.js)
    → discord_api_handler.js  (~300 linii)
        → telegram_ai.js  (MONOLIT — JEDYNE źródło logiki)
            → AI providers (Gemini/NVIDIA)
            → D1 (z nową kolumną platform)
    ← { reply_text, reply_markup }
    ↕ HTTP
Discord API (send message via discord.js)
```

**Kluczowa decyzja**: Discord bot NIE zawiera logiki AI, NIE importuje `telegram_ai.js`, NIE ma własnej kopii kodu. Jest wyłącznie adapterem WebSocket↔HTTP, który:
1. Odbiera wiadomości z Discord Gateway
2. Forwarduje je jako JSON do Worker `/api/discord/webhook`
3. Odbiera `{ reply_text, reply_markup }` z Workera
4. Tłumaczy na discord.js `message.reply()` + `ActionRow`

**Wszystka logika biznesowa, AI, routing, DB — zostaje w `telegram_ai.js` w Workerze.**

### Krok 3: Endpoint `/api/discord/webhook` w Worker

**Nowy plik**: `cloudflare/src/discord_api_handler.js` (~300 linii)

Eksportuje:
- `buildDiscordResponse(replyData)` → `{ reply_text, reply_markup, files? }` w formacie platform-agnostic
- `handleDiscordWebhook(request, env)` → Response

Format platform-agnostic reply_markup (zamiast `inline_keyboard`):

```js
// Telegram zwraca: { inline_keyboard: [[{ text, callback_data }]] }
// Discord otrzymuje: { buttons: [[{ label, action, value? }]] }
// gdzie action = "callback" | "url" | "command"
```

`handleDiscordWebhook`:
1. Weryfikuje `X-Discord-Bot-Secret` vs `env.DISCORD_BOT_SECRET`
2. Parsuje JSON body → standard message format
3. Odczytuje intencję (komenda `!xxx`, issue, konwersacja)
4. Wywołuje istniejące funkcje z `telegram_ai.js` z `platform = "discord"`
5. Tłumaczy `inline_keyboard` → `buttons` w formacie platform-agnostic
6. Zwraca `{ reply_text, reply_markup, files? }`

**Czas**: 3h

**Nowy katalog**: `discord/`

**`discord_bot.mjs`** (~200 linii):

```js
import { Client, GatewayIntentBits } from 'discord.js';
// Brak importów z cloudflare/src/ — bot nie ma logiki AI

const client = new Client({
  intents: [GatewayIntentBits.Guilds, GuildMessages, MessageContent]
});

client.on('messageCreate', async (message) => {
  if (message.author.bot) return;
  
  const resp = await fetch(`${WORKER_URL}/api/discord/webhook`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-Discord-Bot-Secret': DISCORD_BOT_SECRET,
    },
    body: JSON.stringify({
      chat_id: message.channelId,
      user_id: message.author.id,
      message_id: message.id,
      text: message.content,
      username: message.author.username,
      attachments: message.attachments.map(a => ({
        url: a.url, name: a.name, contentType: a.contentType,
      })),
    }),
  });
  
  const data = await resp.json();
  if (data.reply_text) {
    await sendDiscordReply(message, data);
  }
});

client.on('interactionCreate', async (interaction) => {
  if (interaction.isButton()) {
    // Forward customId do Workera jako "callback"
    const resp = await fetch(`${WORKER_URL}/api/discord/webhook`, {
      method: 'POST',
      headers: { ... },
      body: JSON.stringify({
        chat_id: interaction.channelId,
        user_id: interaction.user.id,
        callback_data: interaction.customId,
        type: 'callback',
      }),
    });
    // ...
  }
});

client.login(DISCORD_BOT_TOKEN);
```

## Mapowanie UI: Telegram → Discord

| Telegram | Discord |
|----------|---------|
| `inline_keyboard: [[{ text, callback_data }]]` | `buttons: [[{ label, action: "callback", value }]]` |
| `{ text, url }` w inline_keyboard | `action: "url", value: url` |
| `sendTelegramReply(env, msg, text, markup)` | `message.reply({ content: text, components: buildActionRows(buttons) })` |
| `reply_to_message_id` | Automatyczne przez `message.reply()` |
| `/command` | `!command` (komendy tekstowe) |

## Mapowanie Discord message → standard format

```js
function discordToStandard(message) {
  return {
    chat_id: message.channelId,        // Snowflake string
    user_id: message.author.id,         // Snowflake string
    message_id: message.id,             // Snowflake string
    text: message.content || null,
    file_id: message.attachments.first()?.url || null,  // dla kompatybilności z telegram_ai.js
    caption: message.content || null,
    username: message.author.username,
    mime_type: message.attachments.first()?.contentType || null,
    file_name: message.attachments.first()?.name || null,
    // attachments: do pobierania przez fetchDiscordAttachmentAsBase64
  };
}
```

## Obsługa załączników

`history.js` już zawiera `fetchDiscordAttachmentAsBase64(attachmentUrl)`. W `telegram_ai.js`, tam gdzie kod woła `fetchTelegramFileAsBase64(env, message.file_id)`, dla Discord przekazujemy URL zamiast file_id:

```js
const base64 = platform === "discord"
  ? await fetchDiscordAttachmentAsBase64(message.file_id)  // file_id = URL
  : await fetchTelegramFileAsBase64(env, message.file_id);
```

## Co NIE jest potrzebne (vs oryginalny plan)

- ❌ Kopiowanie logiki AI do `discord_bot.mjs`
- ❌ `discord_message_mapper.mjs` — mapowanie inline w handlerze
- ❌ `discord_ai_bridge.mjs` — nie ma mostu, wszystko w Worker
- ❌ Osobne endpointy REST dla każdej operacji DB — jeden endpoint `/api/discord/webhook`
- ❌ Refactoring standalone modułów (nieużywane, zostają jak są)
- ❌ Dodawanie parametru `platform` do `sessions.js`, `history.js`, `ai_logic.js` — te pliki nie są używane
- ❌ `discord_handlers.mjs` — handler buttonów inline w `discord_bot.mjs`

## Ryzyka

1. **Discord limit 2000 znaków na wiadomość** vs Telegram 4096. Rozwiązanie: worker zwraca `reply_text` do 2000 znaków, dla dłuższych odpowiedzi zwraca `{ reply_text, continuation?: true }` i Discord bot wysyła follow-up.
2. **Discord Embed limit**: 4096 w description, 6000 łącznie. Można użyć Embed dla dłuższych odpowiedzi jako fallback.
3. **Rate limiting**: Discord ma własny rate limiting, discord.js obsługuje go automatycznie.
4. **Timeout Worker CPU (30s)**: AI call może trwać dłużej — ale to istniejący problem dla Telegram, nie nowy.
